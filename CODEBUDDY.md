## 三、核心概念

### 恒市值法（Constant Market Value）
- 每个资产有**固定目标市值**（如黄金始终 ¥100,000），永不改变
- 每月检查：实际市值偏离目标 > ±5% → 调仓回目标
- 现金作为"吸收池"：其他资产卖出回流，买入从中扣除
- **不是恒定比例法**（那个会让目标随总市值涨跌）

### 稳健型配置（当前页面默认）
| 资产 | 目标比例 | 目标市值 |
|------|---------|----------|
| 沪深300 | 15% | ¥75,000 |
| 中证500 | 5% | ¥25,000 |
| 标普500 | 15% | ¥75,000 |
| 纳斯达克100 | 20% | ¥100,000 |
| 黄金 | 20% | ¥100,000 |
| 现金·货币基金 | 25% | ¥125,000 |

### 两种建仓模式
- **一次建仓版**（2015-08 起点，131个月）：第1个月全仓买入到位
- **分批建仓版**（其余10个起点）：12个月逐步建仓

## 四、数据来源

- **月度收益数据**: 新浪财经前复权K线（2015-08 ~ 2026-07，131个月）
- **基金净值**: 天天基金网 API（13只基金）
- **数据处理**: 前复权含分红再投资、多基金规模加权合成指数、QDII溢价3%/8%分档

## 五、关键业务规则

### 颜色规范（中国习惯）
- 🔴 红色 = 上涨/盈利 → `--color-success: #c53030`
- 🟢 绿色 = 下跌/亏损 → `--color-danger: #1a7d3a`
- 三个主题 CSS 文件均已统一

### 回测参数
- 初始资金: 50万
- 再平衡阈值: ±5%
- 交易费率: 0.1%
- 建仓期: 1个月（一次建仓）或 12个月（分批建仓）

### JavaScript 陷阱
- **`||` vs `??`**: 月收益为 `0` 时，`0 || fallback` 返回 fallback（bug！），必须用 `0 ?? fallback`
- **现金管理**: `holdings['现金·货币基金']` 和 `cashBalance` 需在每月初合并

## 七、部署流程

1. `git add -A && git commit -m "..." && git push origin main`
2. Cloudflare 自动检测推送，执行 `npx wrangler deploy`
3. **`wrangler.jsonc`** 声明了 Worker 名称 `hdszf` 和静态资源目录 `.`，确保部署时不会因同名 Worker 冲突而中断
4. 这是 Workers + Assets 模式（不是 Pages），静态文件通过 `assets.directory` 指定
